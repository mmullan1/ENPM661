# ENPM661 – Project 3 Phase 2  
## A* Path Planning with TurtleBot3 in ROS2 + Gazebo

## Overview
This project implements the **A\*** path planning algorithm for a **non-holonomic differential drive robot (TurtleBot3)** and integrates it into a **ROS2 + Gazebo simulation environment**.

The planner generates a collision-free trajectory considering:
- Robot kinematics (differential drive constraints)
- Obstacle space with clearance
- Wheel RPM-based motion primitives

The planned path is then executed in simulation by publishing velocity commands (`/cmd_vel`) to the robot.

---
## Phases
This project is broken down into two spaller phases: 
- Phase 1: Write **A\*** planning algorithm and visualize the explored paths
- Phase 2: Implement this algorithm in both Gazebo and FalconSim

Because of technical difficulties, the FalconSim section is not yet complete, so this ReadMe will document the work done in Phase 1 and the Gazebo portion of Phase 2


---

## Objectives
- Implement A* for a differential drive robot  
- Generate valid motion primitives using wheel RPMs  
- Avoid obstacles with clearance and robot radius  
- Integrate planner into ROS2 node  
- Execute trajectory in Gazebo simulation

## How to Run

### 1. Clone
```
git clone https://github.com/mmullan1/ENPM661.git
```

---

### 2. Navigate to Workspace (Terminal 1)
**In one terminal:**
From the root of the cloned repository (i.e., wherever `ENPM661` was cloned on your system), run:

```
cd ~/ENPM661/Project3/Phase2/Phase2/Gazebo/project3_ws
```

---

### 3. Build Workspace
```
rm -rf build install log
colcon build --packages-select turtlebot3_project3
```

---

### 4. Source Environment
```
source /opt/ros/humble/setup.bash
source install/setup.bash
export TURTLEBOT3_MODEL=waffle
```

---

### 5. Launch Gazebo Environment
```
ros2 launch turtlebot3_project3 competition_world.launch.py
```

---

### 6. Navigate to Scripts (Terminal 2)
**In a separate terminal:**
```
cd ~/ENPM661/Project3/Phase2/Phase2/Gazebo/project3_ws/src/turtlebot3_proj3_661/scripts
```

---

### 7. Run A* Algorithm
```
python3 a_star_michael.py
```

**Example Inputs Used:**
- Start: (100, 0, 90)  
- Goal: (100, 390)  
- Clearance: 2.5 mm  
- RPMs: (50, 100)

---

### 8. Move Robot in Gazebo
```
python3 gazebo_interface.py
```

When prompted, select:
```
astar_path.csv
```

This file is generated by `a_star_michael.py` and contains the trajectory commands sent to the robot. Once the path is complete, a separate
window will pop up displaying how the robot actually moved compared to its desired trajectory. I initially used this as an external
verification to know that my code is working, and I left it in to demonstrate my thought process
